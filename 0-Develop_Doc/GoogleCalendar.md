- 用户在使用`cleck`模块进行登录或注册时，`clerk`支持用新邮箱注册，也支持一键关联谷歌账号，就是直接用谷歌账号登录
- 所以在实现同步到谷歌日历的功能时，我认为需要进行如下的一个逻辑判断：

# 一、用户未登录时

- 当用户点击`calendar sync`按钮时，会先触发付费墙的判定逻辑。

# 二、用户已登录时

## 已登录但`clerk`未与谷歌账号关联

### 非付费用户

- 当用户点击`calendar sync`按钮时，会先触发付费墙的判定逻辑。

### 付费用户

- 用户虽已付费，但是他的账号没有与谷歌账号进行关联，所以此时会按照流程将用户的这个`clerk`账号与谷歌账号进行关联

## 已登录且`clerk`已与谷歌账号关联

### 非付费用户

- 当用户点击`calendar sync`按钮时，会先触发付费墙的判定逻辑。

### 付费用户

- 用户已付费并且账号已与谷歌账号关联，此时将日历视图里的事件卡片信息同步到谷歌日历
