# 二、`cloud save`

- 当pro用户点击`cloud save`按钮时，会弹出一个对话框、模态框，anyway，总之是个交互界面
- 交互界面上会显示，当前已经存储在数据库里的模板，每个模板都有一个独一无二的名字，不准有重名的模板
  - 交互界面上会显示所有当前已经存储的模板。
- 用户可以点击任意一个模板，选择把这个模板下载到本地。
  - 当用户把服务器里的模板下载到本地时，会对本地的local storage进行交校验
    - 如果local storage为空，则直接在日历视图中显示当前模板的日程表
    - 如果local storage不为空，则弹出提示框，提醒用户会清除当前日历视图中已有日程表
- 交互界面上也会有上传的按钮，可以把本地的、用户已经已经在当前的日历视图中创建好的日程表，上传至数据库
- 上传的时候，会要求用户，为日程表起一个独一无二的名字。当用户起好名字后，会把这个名字与当前数据库里已经存在的模板的名字进行校对。确保名的不会重复

# 三、需要确认的问题

### 1. **模板数量限制**

- 无限制的话不太可行，当前先限制10个模板
- 后续根据运营情况可以做调整

### 2. **模板命名**

- 有限制，50个字符可以
- 是否允许中文/特殊字符？
  - 都允许，服务主要面向英语用户，但是不排除以后会有其他语言的用户。

### 3.**已保存模板的操作**

- **重命名**？
  - 需要有这个功能，可以给数据库的模板重命名。
- **删除**？
  - 需要有删除模板的功能
- **查看模板详情**（例如事件数量、创建时间）？
  - 需要有这个功能

### 4. **同名覆盖/更新逻辑**

当用户上传一个模板时，如果输入的名字与已有模板**完全相同**：

- 提醒用户当前要上传的模板和数据库里的模板重名，给用户两个选项，覆盖数据库里的模板或者重命名当前的模板

### 5. **数据结构**

- settings里的设置一定要保存。
