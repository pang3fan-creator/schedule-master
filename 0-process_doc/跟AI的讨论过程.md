# 怎么把我的PRD文档作为context拖进Cursor里？

> 将 PRD 文档作为 Context（上下文）喂给 Cursor 有**两种最常用的方法**，操作非常简单。
>
> 前提：请确保你已经把刚才那份 PRD 内容保存为了一个文件（比如命名为 `PRD.md` 或 `product-requirements.md`），并放在了你的项目根目录下。
>
> ### 方法一：直接拖拽 (最直观)
>
> 1. 在 Cursor 中，使用快捷键 `Cmd + L` (Mac) 或 `Ctrl + L` (Windows) 打开右侧的 **AI Chat** 对话框。
> 2. 在左侧的 **Explorer (资源管理器)** 中找到你的 `PRD.md` 文件。
> 3. 鼠标左键按住该文件，直接**拖拽**到右侧 Chat 对话框的**输入框 (Type a message...)** 里。
> 4. 你会看到输入框里出现了一个小方块，上面写着文件名，这就代表文件已经被“读取”了。
>
> ### 方法二：使用 `@` 符号 (最推荐，效率最高)
>
> 这是 Cursor 的核心功能，以后你会经常用到：
>
> 1. 打开 **AI Chat** (`Cmd/Ctrl + L`)。
> 2. 在输入框里输入 **`@`** 符号。
> 3. 此时会弹出一个下拉菜单，列出你项目里的所有文件。
> 4. 继续输入文件名（例如输入 `prd`），列表会自动过滤。
> 5. 按 **回车** 或鼠标点击选中那个文件。
>
> ------
>
> ### 💡 为什么这样做？
>
> 当你把文件作为 context 放入后：
>
> - **Cursor 会索引全文：** 它会“读懂”你文档里的每一个字（比如你的 5 个长尾词配置、你的配色要求、你的 API 字段定义）。
> - **无需重复 Copy/Paste：** 你不需要把文档内容复制粘贴进去，不仅省事，还能节省 AI 的 Token 上下文窗口（Cursor 对文件引用有特殊的优化）。
>
> **现在的操作步骤：**
>
> 1. 新建文件 `PRD.md`。
> 2. 把刚才生成的文档内容粘贴进去并保存。
> 3. 用上面任意一种方法，把它喂给 Cursor。
> 4. 然后输入指令：“@PRD.md 请帮我生成 `lib/data/templates.ts` 文件...”
